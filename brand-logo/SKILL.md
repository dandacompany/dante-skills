# brand-logo

> "멋진 로고 만들어줘"는 매번 다른 결과를 낸다. 이 스킬은 **브랜드 전략 → 형태 언어 → 제약 → 산출물 형식**을 거쳐 로고 컨셉을 디렉팅하는 패턴이다. 바로 이미지를 만들지 말고, 먼저 브리프를 짜고 프롬프트를 설계한 뒤 생성한다.

## 핵심 전제 (먼저 읽을 것)

- **AI 이미지 모델은 글자에 약하다.** 워드마크(브랜드명 텍스트)는 철자 깨짐·자간 들쭉날쭉이 흔하다. 그래서 **심볼 마크 · 추상 마크 · 모노그램(글자 1개)을 먼저 뽑고**, 완성형 글자·폰트·컬러는 Figma/Illustrator/Canva에서 후처리한다.
- **추상어 금지.** "beautiful·cool·stylish"는 안 먹힌다. **"flat vector · strong silhouette · scalable · favicon-ready · geometric · balanced negative space"** 같은 디자인 조건어로 말한다.
- **생성은 비결정적이다.** 같은 프롬프트도 매번 다르게 나온다. 한 장에 의존하지 말고 방향당 여러 장 뽑아 고른다.
- **브랜드 토큰을 고정한다.** 브랜드에 `DESIGN.md`(또는 디자인 시스템)가 있으면 **먼저 읽고** 그 hex·폰트·비율·금지 목록을 프롬프트에 그대로 박는다. "브랜드 색" 같은 모호어 금지.

## 언제 쓰나

- 새 브랜드/채널/제품의 **로고·심볼·앱 아이콘·파비콘** 컨셉을 만들 때
- 기존 브랜드 디자인 시스템(`DESIGN.md`)에 맞는 **온브랜드 마크**가 필요할 때
- 칸반 카드 등에서 "로고·VI" 임무를 **흔들리지 않게** 수행해야 할 때

## 로고 디렉터 워크플로 (5단계)

### 1. 브리프 — 바로 이미지를 만들지 않는다

브랜드 정보를 받아 **로고 디자인 브리프**를 먼저 정리한다. 항목은 `references/brief-and-eval.md`의 10항목 템플릿을 그대로 쓴다(브랜드 핵심 정체성 / 타깃 첫인상 / 피할 클리셰 / 시각 은유 10개 / 적합한 로고 유형 3개 / 팔레트 3안 / 폰트 방향 3안 / 심볼 형태 방향 5안 / 최종 프롬프트 3개 / 평가 체크리스트).

> 브랜드에 `DESIGN.md`가 있으면 팔레트·폰트·무드·금지를 **새로 만들지 말고 그대로 인용**한다. 토큰은 브랜드의 단일 진실원천이다.

### 2. 컨셉 압축 — 은유 10 → 심볼 방향 3~5

은유 10개를 겹치지 않는 심볼 방향 3~5개로 좁힌다. 텍스트 리스크 순으로 우선순위: **추상 마크 > 모노그램(글자 1개) > 워드마크(전체 글자)**. 시연·빠른 확정이 목적이면 글자 적은 쪽을 먼저 뽑는다.

### 3. 프롬프트 작성 — 공식으로 3~5개

프롬프트 공식: **브랜드 정체성 + 업종 + 고객층 + 키워드 + 심볼 방향 + 형태 스타일 + 색상(정확한 hex) + 금지 요소 + 출력 조건**. 영문/한국어 완성 템플릿과 4방향(미니멀 기하 / 프리미엄 추상 / 친근 모던 / 강한 테크) 변형은 `references/prompt-templates.md`. 네거티브 프롬프트는 항상 붙인다(`references/negative-and-craft.md`).

### 4. 생성 — 이미지 도구로 방향당 여러 장

준비된 프롬프트를 이미지 생성 도구로 돌린다. 헤르메스에서는 빌트인 `image_generate`(gpt-image-2 등, 프로필 config의 `image_gen.provider`), Claude 환경에서는 `kie-image-generator` 등. 정사각(1:1) 기본, 흰/투명·파치먼트 등 단색 배경, 플랫 벡터 지향. **방향당 2~4장** 뽑아 비교한다.

### 5. 선별·인계 — 평가 후 벡터화로 넘긴다

`references/brief-and-eval.md`의 평가 체크리스트(단순·확장성·실루엣·여백·온브랜드·텍스트 가독)로 고른다. 확정 후보는 **벡터화(Figma/Illustrator/Canva)** 로 인계하고, 글자·폰트·컬러 시스템은 거기서 정리한다. 철자·브랜드 hex는 사람이 최종 확인한다.

## 절대 제약

- **바로 이미지부터 만들지 않는다.** 브리프·프롬프트 설계가 먼저다.
- **추상어 대신 디자인 조건어.** flat vector · strong silhouette · scalable · negative space · geometric.
- **강한 네거티브 항상 첨부.** no 3D, no mockup, no photo, no gradient, no glow, no shadow-heavy, no stock-logo, no generic icon, no tiny details, no unreadable text, no watermark.
- **텍스트 최소.** 워드마크는 철자 깨짐을 전제하고, 글자는 후처리로 미룬다. 모노그램/추상 우선.
- **브랜드 토큰 고정.** `DESIGN.md`가 있으면 hex·폰트·비율·금지를 그대로. 없으면 브리프 2단계에서 팔레트를 먼저 확정하고 프롬프트에 박는다.
- **여러 장에서 고른다.** 비결정성을 인정하고 한 장에 의존하지 않는다.
- **단일 강조색·강한 실루엣·확장성**(파비콘·앱 아이콘·프로필까지) 을 모든 안에 요구한다.

## 직무 인계

선별된 로고 후보와 평가 노트를 디자이너/브랜드 덱 제작자에게 넘긴다. 벡터 확정본과 컬러·폰트 시스템은 브랜드 디자인 시스템(`DESIGN.md`)에 다시 반영해 단일 진실원천을 갱신한다.
