# report-evidence-citation

> 모든 산출물의 사실/의견 분리 + 출처 URL 보존 + 평가 어휘 차단. 회사 공통 게이트.

## 인용 규칙

- **모든 정성 인용**: `source_url` + `captured_at` + `provenance_note` 보존
- **모든 정량 시그널**: 원본 SQL / 계산식 + 입력 데이터 범위 보존
- **본문 인용 시**: `(출처: <url>)` 형식. 인용 출처 5개 이내 (그 이상은 별도 부록)
- **이미지/차트**: 캡션에 데이터 출처 + 수집일자 명시

## 사실 / 의견 분리

| 구분 | 라벨                             | 예시                                                                           |
| ---- | -------------------------------- | ------------------------------------------------------------------------------ |
| 사실 | (출처: ...)                      | "무신사 베스트 상위 50건 평균가 89,000원 (출처: musinsa.com 랭킹, 2026-05-21)" |
| 의견 | "관찰" / "주목" / "추정"         | "관찰 — 검색 트렌드가 11월에 피크를 보임. 시즌성 가능성."                      |
| 금지 | "추천", "매수", "좋다", "나쁘다" | (사용 금지)                                                                    |

## 보고 형식

- **코멘트 첫 줄**: `사용 도구: <도구 목록>` (예: `사용 도구: Bright Data Web Unlocker + SERP API`)
- **정정 발생 시**: `정정 항목 N건 · 신뢰도 0~1.0` 명시
- **결과 제출**: 반드시 paperclip API 코멘트로 POST (`POST /api/issues/{id}/comments`)

## 검증 체크리스트

- [ ] 본문 모든 수치에 출처 URL 또는 시그널 값이 붙어 있는가?
- [ ] 평가성 어휘("추천"/"매수"/"좋다"/"나쁘다") 0건인가?
- [ ] 의견 라인은 "관찰"·"주목"·"추정" 라벨로 시작하는가?
- [ ] API 키 평문 노출 0건인가? (첫 8자 + `***` 마스킹)
- [ ] paperclip API 코멘트 POST 완료했는가?

위 5개 중 하나라도 위반 시 결과 보고 거부하고 자가 수정.

## 적용 범위

Brand Intelligence Lab 의 **모든 에이전트** 가 상속 (브랜드 리서처 + 데이터 분석가 + 슬라이드 제작자).
